# Contributing

Read the README, About, and Roadmap before contributing.

This project is intended for systems and networks the user is authorized to configure. Use an Issue before making a large architecture, format, validation, or compatibility change.

Keep pull requests focused and include the problem, expected result, validation steps, supported environment, compatibility impact, and documentation updates.

Use clear commit prefixes such as `feat:`, `fix:`, `docs:`, `test:`, `refactor:`, and `chore:`.

Contributors are responsible for correctness, licensing, and review. Do not publish private network information or material that you do not have permission to distribute.
